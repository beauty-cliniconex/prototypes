# Prototypes

> A Figma-style prototype workspace — organized by project folders with versioned subfolders.
>
> ---
>
> ## How it works
>
> Each **folder = a project**. Inside each project, each **subfolder = a version**. Add your prototype files (HTML, assets, etc.) inside the version folders.
>
> ```
> prototypes/
> ├── project-alpha/
> │   ├── v1/         ← Version 1
> │   └── v2/         ← Version 2 (updated iteration)
> ├── project-beta/
> │   └── v1/         ← Version 1
> └── README.md       ← You are here
> ```
>

>
> ## How to add a new project
>
> 1. Create a new folder: `your-project-name/v1/`
> 2. 2. Add a `README.md` describing the project
>    3. 3. Add your prototype files (`index.html`, assets, etc.)
>      
>       4. ## How to add a new version
>      
>       5. 1. Duplicate your latest version folder (e.g. copy `v2/` → `v3/`)
>          2. 2. Update the files with your new changes
>             3. 3. Update the table above with the new version
>
